2019-02-19 21:21 UTC-0300 Fernando Yurisich <fyurisich@oohg.org>
   * ide\build.bat
     * INCOMPATIBLE: HB30/HB32 parameters changed to HM30/HM32.
       H now means Harbour and M means MinGW.
     + Support for Viktor Szakats Harbour 3.4 + MinGW clang
       compiler (HM34).
   * ide\compile.bat
     + Support for xHarbour + MinGW gcc compiler (XM).

   * ide\prgs\formedit.prg
     ! Some properties are not copied or swapped when a control is
       copied or it's order inside a tab is changed.
     ! Red lines around the focused control are not erased when
       the form is scrolled.
     ! Tool forms are not shown at startup if they were minimized
       on exit.
     ! Control's move and size.
     ! Wrong toolbar's height.
     ! Buttons can't be resized using the mouse.
     ! The names of some controls are forcibly uppercased.
     ! Some clauses are not recognized when their not uppercased.
     ! Many clauses were wrongly enclosing it's parameters with
       quotes thus preventing the use of variables in clauses.
       ; To work properly literal values must be quoted.
       ; Unquoted values will be output as is.
     + Clauses added since last update and some others missing.
     + Support for nested TAB controls.
     + Support for #define and #undef constants. Note that, in
       order to be processed, a #define must be placed before the
       DEFINE WINDOW TEMPLATE clause, and an #undef must be placed
       after the END WINDOW clause. In between directives will be
       ignored. These only applies to DEFINE WINDOW properties.
     + Support for color constants in FONTCOLOR and BACKCOLOR
       clauses.
     + Form's "control order" functionality can now move controls
       between the pages of a TAB.
     + A context menu option to restore minimized form tools.
     + Some checks to ON KEY processing logic to avoid information
       looses when a data edition results in an invalid array.
       ; The 'key' and the 'action' parts can't contain, at the
         same time, a quotation mark ("), an apostrophe (') and a
         closing square bracket (]).
       ; Also they can't contain an opening brace ({) or and
         ending brace (}).
       ; If you need to use these symbols you must use a #define
         constant to substitute the offending part.
     * Methods SwapArray, AddArray, DelArray now use macros over an
       array to simplify the addition of new properties and reduce
       the chances of repeating the aforementioned error.
     * Statusbar related buttons were swapped: the on/off button is
       now at the controls toolbar and the properties control is
       now at the designer toolbar.
     * A red filled box is now displayed in all cases when moving
       or sizing a control in order to show the progress.
     % Several minor optimizations.
     % Code readability and comments.
